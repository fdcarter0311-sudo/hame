# Codex Master Brief for Eidolon Drift Project

## Project Overview
The Eidolon Drift project is a unique and immersive gaming experience designed to explore themes of exploration, identity, and alternate realities. Players will navigate through a beautifully crafted world filled with secrets, challenges, and narratives that shift based on their choices. The primary goal is to create a game that is not only engaging but also emotionally resonant, with a strong focus on player agency and storytelling.

## MVP Scope
- **Core Gameplay Loop**: Define and implement the fundamental interactive experiences.
- **Basic World Design**: Create initial environments that convey the game's themes and narratives.
- **Character Development**: Design main character attributes and skill trees.
- **Narrative Structure**: Establish a branching story path with key decisions.

## Core Systems
1. **Player Control and Mechanics**: Develop intuitive controls that allow fluid movement and interaction.
2. **World Interaction**: Build systems for environmental interaction, including puzzles and discovery.
3. **Inventory Management**: Implement a system for item collection and usage.
4. **Dialogue System**: Create a branching dialogue system for player interactions with NPCs.
5. **Save/Load Mechanism**: Ensure players can save progress at key points in the game.

## Milestones
1. **Milestone 1: Concept Art and Prototyping** (Date: 2026-05-15)
   - Completion of initial concept art.
   - Prototype of core mechanics completed.
2. **Milestone 2: Vertical Slice** (Date: 2026-06-30)
   - A small playable section of the game that showcases visuals, mechanics, and narrative.
3. **Milestone 3: Alpha Release** (Date: 2026-09-01)
   - Complete story and mechanics.
   - Begin internal testing.
4. **Milestone 4: Beta Release** (Date: 2026-12-01)
   - Launch of beta version for external feedback.
5. **Milestone 5: Final Release** (Date: 2027-04-01)
   - Full game launch.

## Technical Architecture
- **Game Engine**: Unity3D or Unreal Engine (to be decided based on staff skill).
- **Networking**: Implement required services for online interactions if needed.
- **Art Pipeline**: Define and document the workflow for art assets from concept to integration.
- **Sound Design**: Specify requirements and guidelines for audio assets.

## Skill Definitions
- **Game Designer**: Responsible for gameplay mechanics, level design, and narrative.
- **Artist**: Handles visual assets, including 2D and 3D graphics.
- **Developer**: Implements game logic, systems, and features.
- **Sound Designer**: Creates and integrates sound effects and music.
